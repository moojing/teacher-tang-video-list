# 唐師父直播主題索引設計規格

## 目標

以 `/Users/mujingtsai/Downloads/唐師父直播主題索引.html` 為視覺基準，建立一個 React 靜態影片索引頁。第一階段不連接 Notion API，影片資料放在專案內的 JSON，讓畫面與互動先可以獨立完成；未來可在不改動主要 UI 的前提下替換資料來源。

## 範圍

包含：

- 深色星象主視覺、星空背景、標題與說明
- 主題環形導覽，顯示熱門主題與影片數量
- 影片標題搜尋
- 會員狀態篩選
- 主題標籤篩選
- 依日期或標題排序
- 影片清單、日期、標籤、結果數量與無結果狀態
- 點擊影片項目開啟 YouTube 連結
- 桌面與手機版響應式排版

不包含：

- Notion API、API token 或任何伺服器端資料同步
- 登入、會員權限驗證與播放權限判斷
- 後台編輯影片資料
- YouTube 內嵌播放器

## 技術方案

使用 Vite + React + JavaScript。樣式採用專案自己的 CSS，不引入 UI framework，因為原稿的星象視覺、環形導覽與標籤系統需要較細緻的自訂樣式。

資料放在 `src/data/videos.json`，欄位統一為：

```json
{
  "id": "stable-video-id",
  "title": "影片標題",
  "date": "2026-07-18",
  "url": "https://www.youtube.com/watch?v=...",
  "topics": ["主題"],
  "status": "一般公開"
}
```

日期允許為 `null`。`id` 用於 React list key；如果來源資料沒有穩定 ID，匯入靜態資料時由影片 URL 或資料索引產生。

## 元件結構

```text
App
├── Hero
│   └── TopicOrbit
├── VideoExplorer
│   ├── SearchField
│   ├── StatusFilters
│   ├── TopicFilters
│   ├── SortControl
│   └── VideoList
│       └── VideoItem
└── Footer
```

`App` 只負責組合頁面與管理篩選狀態。`VideoExplorer` 取得完整資料並衍生篩選後的結果；篩選與排序函式集中在 `src/lib/video-utils.js`，避免把資料邏輯散落在展示元件中。

## 狀態與互動

頁面狀態：

- `query`: 搜尋字串，搜尋影片標題與主題
- `activeStatus`: 目前狀態；預設為全部
- `activeTopic`: 目前主題；預設為全部
- `sortBy`: 預設日期新到舊

互動規則：

1. 搜尋、狀態與主題篩選使用 AND 組合；同時符合條件的影片才會顯示。
2. 搜尋不分大小寫，並移除前後空白；空字串代表不套用搜尋條件。
3. 日期排序以有效日期為優先，沒有日期的項目固定排在最後。
4. 點擊目前已選中的主題或狀態會回到全部。
5. 點擊影片項目以新分頁開啟其 YouTube URL，並使用 `rel="noreferrer"`。
6. 找不到結果時顯示清楚的空狀態，不保留空白清單區域。

## 視覺與響應式規則

保留原稿的核心視覺 token：

- 背景：`#10152B`
- 面板：`#171D38`
- 金色強調：`#C9A44C`
- 青綠輔助：`#3F7169`
- 主要文字：`#EDE6D6`
- 次要文字：`#8B93B0`
- 分隔線：`#2A3160`

字體沿用原稿的中文與英文字體組合。桌面版採中央限寬內容區；手機版將篩選列改為可水平滾動的標籤群組，影片項目改為單欄排列，環形主題導覽縮小但保留可點擊性。所有固定格式控制項使用穩定尺寸，避免標籤文字造成版面跳動。

不加入額外的卡片巢狀、漸層球體或行銷式區塊；影片清單維持適合掃讀的單層項目排列。

## 資料匯入

從設計稿內的 `DATA` 陣列提取靜態資料至 JSON，保留原有影片名稱、日期、YouTube URL、主題與狀態。資料匯入後以腳本或一次性檢查確認 JSON 可被解析，且每筆資料具有必要欄位。

未來接 Notion 時，新增一個資料來源 adapter，將 Notion page properties 轉成上述資料模型；UI 只依賴標準化後的 `Video` 資料，不直接讀取 Notion 欄位格式。

## 驗證

實作完成後驗證：

- 安裝依賴、正式建置成功
- 搜尋、狀態篩選、主題篩選與排序互動正確
- null 日期不會造成排序或顯示錯誤
- 空結果狀態可正常顯示與恢復
- 桌面與手機寬度沒有水平溢出或文字重疊
- 主要影片連結存在且可點擊
- 使用瀏覽器檢查首屏與清單區域的視覺結果
